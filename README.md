# Discretka — тесты простоты, визуализации и отчёты (КР + ПР)

Репозиторий с кодом для генерации данных, построения визуализаций и автоматической сборки Markdown-отчётов по дискретной математике: как для **контрольной работы**, так и для **практических**.

```
.
├─ out_dm/                  # Итоги по контрольной работе
│  ├─ report_discrete_dm.ipynb
│  └─ report_discrete_dm.md
├─ out_pr6/                 # Итоги по практической работе 6
│  ├─ out_files/            # Рисунки/вложения для отчёта
│  ├─ pr6_report_all.ipynb
│  └─ pr6_report_all.md

├─ pr6.py                   # Скрипт с заданиями/генерацией данных для ПР 6
├─ pr6_build_report.py      # Сборка итогового отчёта по ПР 6
├─ task1_text_coding.py     # Задание 1 (текстовое кодирование)
├─ task2_hamming.py         # Задание 2 (код Хэмминга)

```

## Быстрый старт

Рекомендуется отдельное виртуальное окружение:

```bash
python -m venv .venv
# Windows:
.venv\Scripts\activate
# Linux/macOS:
source .venv/bin/activate

# Если есть requirements.txt:
pip install -r requirements.txt
```

> Если `requirements.txt` отсутствует: скрипты используют стандартную библиотеку Python и jupyter/ipykernel для `.ipynb`. Установи при необходимости:
> ```bash
> pip install jupyter ipykernel
> ```

## Как запустить

### Вариант 1. Универсальный вход
```bash
python main.py
```
Этот скрипт может вызывать нужные пайплайны (КР/ПР) — см. аргументы, если они предусмотрены в коде.

### Вариант 2. Контрольная работа (КР)
```bash
python dm_main.py
```
Результаты появятся в `out_dm/`:
- `out_dm/report_discrete_dm.md` — итоговый отчёт (Markdown)
- `out_dm/report_discrete_dm.ipynb` — ноутбук (если используется)

### Вариант 3. Практическая работа (ПР)
1) Выполнить генерацию/расчёты (при необходимости):
```bash
python pr6.py
```
2) Собрать единый отчёт:
```bash
python pr6_build_report.py
```
Результаты — в `out_pr6/`:
- `out_pr6/pr6_report_all.md` — итоговый отчёт по ПР
- `out_pr6/pr6_report_all.ipynb` — ноутбук
- `out_pr6/out_files/` — рисунки и встраиваемые ресурсы

### Отдельные задания
```bash
python task1_text_coding.py   # Задание 1: текстовое кодирование
python task2_hamming.py       # Задание 2: код Хэмминга
```

## Содержание отчётов (в общем виде)

- Классические вероятностные **тесты простоты**: Ферма, Соловей–Штрассен, Миллер–Рабин.
- **Псевдопростые** и **числа Кармайкла**, объяснение ограничений теста Ферма.
- Аналитика по простым числам (плотность, разрывы) и **наглядные визуализации**.
- **Профили времени**: сравнительная производительность тестов на одной и той же выборке.
- Краткие **выводы** к каждому рисунку/разделу.

> Если в коде используются встроенные помощники разметки (например, генерация подписи вида:
> «Ось X — основание *a*, ось Y — нечётные модули *n* (3..199). Цвета: +1, 0, −1. Нули там, где gcd(a,n)>1. Мозаика ±1 демонстрирует квадратичную взаимность…»),
> эти комментарии автоматически попадают под соответствующие графики и поясняют, **зачем** добавлены доп. методы/визуализации.

## Рекомендации по использованию тестов

- Для 64-битных целых используйте **Миллер–Рабин** с известными наборами оснований — на практике это детерминированно и быстро.
- Для больших *n* повышайте число раундов *t*: вероятность ложноположительного результата падает близко к экспоненциальной.
- **Ферма** — только как быстрый «скрининг»; обязательно дополнять MR/SS.
- **Соловей–Штрассен** полезен для анализа (через символ Якоби) и отбрасывает много составных.

## Повторяемость экспериментов

- Используйте фиксированные `seed` (если предусмотрены в коде) — это сделает отчёты воспроизводимыми.
- Тяжёлые операции (списки простых, факторизации) целесообразно кэшировать.
